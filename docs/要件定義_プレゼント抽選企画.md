## 0. 現行サイトの構成(前提知識)

- 静的サイト(index.html / styles.css / script.js)。Cloudflare Workers 上で公開
- 企画: GENGEN vs エンター豊田のチャリティーゴルフ対決(7/28、マッチプレー、生配信)。生配信の投げ銭は手数料等を除き全額寄付(GENGEN陣営=児童養護施設、エンター豊田陣営=保護犬・保護猫)

- 両者のSNS: GENGEN(Instagram @gengen_ch / YouTube / TikTok @gengen.ch)、エンター豊田(Instagram @tatsuhiro.toyoda / YouTube / TikTok @entoyoyo)。リンクはLP掲載済み

## 1. 企画の主従関係(最重要の設計方針)

**主役はチャリティ。プレゼントは参加特典。**

LPの訴求優先順位:
1. チャリティーマッチ — それぞれが応援する団体への想いを胸にした対決
2. 生配信で応援・投げ銭 — 視聴者の応援が寄付につながる
3. 事前応援投票 — どちらの陣営を応援するか参加できる
4. プレゼント抽選 — 投票参加者向けの**参加特典**(主役にしない)

コピーの温度感:
- LPの見出しは「投票参加者限定プレゼント」「応援参加ありがとうプレゼント」程度に抑える
- 「GENGENが負けたらパター、エンター豊田が負けたらドライバー」という強いフックは**LPでは前面に出さず、SNS投稿(リール/ショート動画)側で使う**。SNSでは釣り力が機能し、踏んだ先のLPはチャリティのサイトとして受け止められる

## 2. プレゼントのルール(確定)

- マッチプレーの勝敗に応じてプレゼントを提供
  - エンター豊田が負けたら → ドライバー
  - GENGENが負けたら → パター
- 賞品は試合結果で決まる1種類。投票後にプレゼント抽選へ応募した方の中から抽選
- LP上のサブコピー例: 「勝敗によって賞品が決定。GENGENが負けたらパター、エンター豊田が負けたらドライバーを抽選でプレゼント。」

## 3. 応募・抽選の仕様

### 3-1. 応募フロー(v5: 毎日応募・モーダル統合型)

1. LPで陣営の応援ボタンを押す → 投票は従来どおり即送信(匿名・1日1回)
2. **投票完了モーダルの中に毎回そのまま応募フォームが表示される**
   - 応援した陣営は押したボタンから**自動セット**
   - 入力: メールアドレス(必須)/ ニックネーム(必須)/ 当選率UPチェック(任意)
   - 応募しない人は「閉じる」だけ(投票は完了している)
3. 送信 → モーダル内で「本日の応募完了」を表示
4. 翌日は再び投票でき、投票後の応募で抽選口数がさらに1口増える
5. 同じメールアドレス・同じ日付の重複応募は集計上1口として扱う
6. 投票の気軽さ(ワンクリック・匿名)を維持し、毎日の投票と応募をリピート動機にする

### 3-2. 抽選口数(現行リリース仕様)

| アクション | 口数 | 確認方法 |
|---|---|---|
| 投票後の当日応募 | 1日につき+1口 | メールアドレス+応募日で集計 |
| GENGENのTikTokフォロー | 期間中1回のみ+1口 | 自己申告 |
| エンター豊田のYouTubeチャンネル登録 | 期間中1回のみ+1口 | 自己申告 |

合計抽選口数は「有効な応募日数+フォロー・登録ボーナス最大2口」。フォーム上は各項目の横に該当ページへのリンクを置き、その場でフォロー/登録できるようにする。
チャンネル登録・フォローの自動検証はAPI上不可能なため自己申告とし、「当選連絡時に確認する場合があります」と注記。

### 3-3. 実装方式

- 応募フォームは現行投票と同じ「新規Googleフォームへ裏側POST(no-cors)」方式。ページ遷移なし・無料・スプレッドシートに自動集計
- プレゼント応募用Googleフォームと各entry IDは接続済み
- メール形式バリデーションはサイト側で実施
- 回答シートとは別に「抽選集計」タブを用意し、メールアドレスごとに応募日数・各ボーナス・合計抽選口数を自動集計
- 同じメールアドレスから同日に複数回答があっても、毎日応募口数は1口として数える

## 4. 拡散設計(Instagram / YouTube / TikTok)【一旦保留】

Xの「リポスト=グラフ拡散」は使えないため、**エンゲージメントを集めておすすめ面(発見タブ/FYP/ショートフィード)に乗せるアルゴリズム拡散**+**ストーリーズによる友人グラフ拡散**で設計する。

1. **本人発信が拡散の主エンジン**(サイト外・必須): 両者がリール/TikTok/ショートで「負けたらこれを誰かにあげます」と実物を見せる動画を各1本以上。強いフックはここで使う。概要欄・コメント固定・プロフィールリンクにLPのURL
2. **応援コメント企画**(必須): 予告動画への応援コメントを抽選口数に組み込む(3-2)。コメントが増える→エンゲージメント上昇→おすすめ露出増、という好循環を応募インセンティブで作る
3. **ストーリーズシェア+メンション**(必須): Instagramで唯一「他人のタイムラインに露出する」行為。+2口と厚めに設定。メンションは運営に通知が届くため実質検証可能
4. **共同目標型の追加賞品**(任意): 「応援総数500票超えで勝者からもサイン入りグッズ追加」など。既存の応援メーターと連動させ、シェアする理由を全員に作る

1. **ヒーロー**: 現行の「応援が寄付につながる」軸を維持・強化。プレゼントは投票セクションで案内

3. **投票完了モーダル改修**: 応募CTAを追加
4. **応募フォームUI追加**(LP内、Googleフォーム裏POST)
5. **文言差し替え**: 「※投票は匿名です。個人情報は一切取得しません」→「投票は匿名です。プレゼント応募時のみ、抽選・当選連絡のためメールアドレスとニックネームをいただきます」+利用目的明記(抽選・連絡のみに使用、企画終了後削除)
6. **OGP更新**: description をチャリティ主軸+事前応援投票の順で更新
